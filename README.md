# Block Bloom

Three.js와 TypeScript로 구현한 웹 기반 1인칭 복셀 샌드박스 게임입니다. 서버나 별도의 게임 엔진 없이 브라우저에서 절차적으로 생성된 섬을 탐험하고 블록을 설치하거나 제거할 수 있습니다.

## 프로젝트 소개

Block Bloom은 복셀 월드의 생성, 렌더링, 충돌 처리 및 상태 저장을 브라우저 환경에서 구현한 프로젝트입니다. 동일한 시드는 항상 같은 지형을 생성하며, 원본 지형과 달라진 블록만 별도로 기록해 저장 데이터의 크기를 줄입니다.

월드는 16 × 16 크기의 청크로 분할됩니다. 블록이 변경되면 해당 청크와 경계에 인접한 청크의 메시만 다시 생성해 전체 월드를 매번 재구성하지 않습니다.

## 주요 기능

- 시드 기반의 64 × 32 × 64 복셀 월드 생성
- 높이 맵과 규칙 기반의 섬, 해변 및 나무 생성
- 잔디, 흙, 돌, 모래, 나무, 잎, 벽돌 등 9종의 블록
- 1인칭 시점 이동, 점프, 중력 및 AABB 충돌 처리
- 레이캐스팅을 이용한 블록 설치·제거
- 청크 단위 메시 생성 및 변경 영역 갱신
- 브라우저 `localStorage`를 이용한 월드와 플레이어 위치 자동 저장
- JSON 파일을 통한 월드 백업 및 복원
- 저장 데이터 형식과 블록 좌표 유효성 검증

## 실행 방법

### 요구 사항

- Node.js 20 이상 권장
- npm
- WebGL과 Pointer Lock API를 지원하는 최신 데스크톱 브라우저

### 로컬 개발 서버

```bash
git clone https://github.com/luminen529/Lu-Minecraft.git
cd Lu-Minecraft
npm install
npm run dev
```

Vite가 출력한 로컬 주소에 접속한 뒤 **월드 시작** 버튼을 선택하면 게임이 시작됩니다.

### 테스트 및 프로덕션 빌드

```bash
npm test
npm run build
```

`npm run build`는 TypeScript 타입 검사를 수행한 후 `dist/` 디렉터리에 정적 배포 파일을 생성합니다.

## 조작법

| 입력 | 동작 |
| --- | --- |
| `W` `A` `S` `D` | 이동 |
| `Space` | 점프 |
| `Shift` | 빠르게 이동 |
| 마우스 이동 | 시점 조작 |
| 왼쪽 클릭 | 블록 제거 |
| 오른쪽 클릭 | 블록 설치 |
| `1`~`9` / 마우스 휠 | 사용할 블록 선택 |
| `Esc` | 메뉴 열기 |

현재 버전은 키보드와 마우스를 사용하는 데스크톱 브라우저에 최적화되어 있습니다.

## 저장 방식

최초 지형은 시드로 다시 생성할 수 있으므로 저장 파일에는 시드, 플레이어 위치, 원본 지형과 달라진 블록만 포함됩니다. 플레이 중인 월드는 브라우저에 자동으로 저장되며, 메뉴에서 JSON 파일로 내보내거나 기존 파일을 가져올 수 있습니다.

## 프로젝트 구조

```text
src/
├── main.ts        # Three.js 장면, 입력, 이동 및 블록 편집
├── world.ts       # 절차적 지형 생성과 월드 상태 관리
├── storage.ts     # 자동 저장과 JSON 가져오기/내보내기
├── world.test.ts  # 월드 생성 및 저장 데이터 테스트
└── style.css      # 게임 UI 스타일
```

## 배포

`.github/workflows/deploy.yml`은 `main` 브랜치에 변경 사항이 푸시되면 테스트와 프로덕션 빌드를 실행한 뒤 결과물을 GitHub Pages에 배포합니다. 저장소의 **Settings → Pages → Source**가 **GitHub Actions**로 설정되어 있어야 합니다.
